### P1-2 기존 코드 lovely_stock_list 사용 위치 파악 ✅

#### 확인 결과

| 파일 | 참조 수 | 주요 용도 |
|------|--------|---------|
| `turtle_order_logic.py` | 15곳 | `if code not in lovely_stock_list` 안전장치, 종목명 조회, 진입/피라미딩 목록 반복 |

| `target_manager.py` | 4곳 | `unheld_codes = [code for code in lovely_stock_list ...]` 미보유 목록 생성, 종목명 조회 |
| `sector_cache.py` | 4곳 | 테마 캐시 갱신 대상 목록, 종목명 조회 |
| `config.py` | 2곳 | 상수 정의, `get_stock_name()` 함수 |

**핵심 패턴 3가지 (교체 방법):**
```
패턴①: if code not in lovely_stock_list  →  if code not in get_watchlist()
패턴②: for code in lovely_stock_list     →  for code in get_watchlist()
패턴③: lovely_stock_list.get(code, {}).get("name", code)
        →  get_watchlist().get(code, {}).get("name", code)
```

**sector_cache.py 추가 고려:**
- 현재: lovely_stock_list 5개 종목에 대해서만 테마 캐시 갱신
- 변경 후: 동적 50개 대상으로 갱신 필요
- 단, 매번 50개 × t1532 호출은 부담 → 당일 캐시가 없는 종목만 갱신하는 기존 로직 활용 가능
