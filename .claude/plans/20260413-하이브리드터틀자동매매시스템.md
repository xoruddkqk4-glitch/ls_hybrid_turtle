## 🎯 목표

LS증권 API를 이용해서 국내 주식(삼성전자 등 7개 종목)을 자동으로 사고파는 프로그램을 만든다.

유지되는지 확인한 뒤에만 매수하는 "하이브리드" 전략을 자동으로 실행한다.

| 용도 | API 코드 | 호출 방법 | 응답 필드 |
|------|---------|---------|---------|
| 로그인 | - | `LS().login(appkey, secret, paper_trading)` | True/False |
| 현재가 조회 | t1101 | `.market().t1101(T1101InBlock(shcode)).req()` | `.block.price` |

**[중요 발견사항]**
- 모의투자 주문 시 `IsuNo`는 반드시 `"A" + 종목코드` 형식 사용 (예: `"A005930"`)
- 실계좌 주문 시 `IsuNo`는 종목코드 그대로 (예: `"005930"`)
- 시장가 주문: `OrdprcPtnCode="03"`, `OrdPrc=0`
- 매수: `BnsTpCode="2"`, 매도: `BnsTpCode="1"`

**[환경변수 (.env) 항목]**
- `LS_APP_KEY` — LS증권 앱키
- `LS_APP_SECRET_KEY` — LS증권 앱 시크릿키
- `LS_PAPER_TRADING` — 모의투자 여부 (True/False)
- `LS_ACCOUNT_NO` — 계좌번호 (마스킹 처리용)
- `TELEGRAM_BOT_TOKEN` — 텔레그램 봇 토큰
- `TELEGRAM_CHAT_ID` — 텔레그램 채팅 ID
- `GOOGLE_SERVICE_ACCOUNT_JSON` — Google 서비스 계정 JSON 경로
- `GOOGLE_SPREADSHEET_TITLE` — Google Sheets 시트 제목
- `GOOGLE_DRIVE_FOLDER_ID` — (선택) Google Drive 폴더 ID

### ⑥ target_manager.py — 동적 목표가 관리

**`unheld_stock_record.json` 구조:**
```json
{
  "005930": {
    "pending_target": 75000,
    "above_target_since": "2026-04-13 10:30:00",
    "last_updated": "2026-04-13 10:00:00"
  }
}
```
> `above_target_since`가 null이면 아직 목표가 미달, 값이 있으면 그 시각부터 30분 카운트 시작

| 함수 | 하는 일 |
|------|---------|
| `load_unheld_record()` | unheld_stock_record.json 읽기 |
| `save_unheld_record(record)` | JSON 저장 |
| `calc_pending_target(code, current_price, indicators)` | `max(현재가×1.02, 240분봉20MA×1.005)` 계산 |
| `update_above_target_time(code, current_price, record)` | 현재가 ≥ pending_target이면 타이머 시작/유지, 아니면 초기화 |
| `run_update()` | 전체 미보유 종목 목표가 갱신 (메인 실행 함수) |


### P2-3 프로젝트(코드) 구조 작성 ⬜

> 목적: 실제로 만들 파일과 폴더 구조를 확정하고, 각 파일의 첫 뼈대(틀)를 잡는다.

#### 최종 파일 구조

```
ls_hybrid_turtle/
│
├── [SA-FOUNDATION] ────────────── 기반 모듈 (다른 모든 파일이 의존)
│   ├── config.py                  관심 종목 상수 정의
│   ├── ls_client.py               LS증권 API 전담 래퍼
│   ├── indicator_calc.py          ATR, MA, 10일 신저가 계산
│   ├── trade_ledger.py            체결 원장 (JSON + Google Sheets)
│   └── telegram_alert.py          텔레그램 알림
│
├── [SA-MODULE-ENTRY] ──────────── 진입 검증 모듈
│   ├── target_manager.py          동적 목표가 산출 및 타이머 상태 관리
│   └── timer_agent.py             30분 가드 체크
│
├── [SA-MODULE-TRADE] ──────────── 거래·감시 모듈
│   ├── turtle_order_logic.py      Unit 계산, 진입·피라미딩 주문
│   └── risk_guardian.py           2N 손절·트레일링 스탑 감시
│
├── [실행기 & 설정] ────────────────
│   ├── run_all.py                 통합 배치 실행기
│   ├── .env                       환경변수 (커밋 금지)
│   ├── .env.example               환경변수 작성 템플릿
│   ├── requirements.txt           라이브러리 목록

│
└── [런타임 생성 파일] ──────────── 프로그램이 실행 중에 자동 생성
    ├── unheld_stock_record.json   미보유 종목 상태 (목표가·타이머)
    ├── position_state.json        보유 포지션 상태 (Unit·손절가)
    └── trade_ledger.json          체결 원장 로컬 저장
```

#### 각 파일 역할 설명 (P2-2 보완 5개 반영 완료)

**config.py**
- `lovely_stock_list` 딕셔너리 상수 (종목코드 → 이름·시장)

**ls_client.py**
- `login()` — 모의/실계좌 로그인 (환경변수로 결정)
- `get_multi_price(codes)` — t8407 멀티현재가 (7종목 한 번에)
- `get_daily_chart(code, count=25)` — t8451 일봉 OHLCV
- `get_minute_chart(code, minute=240, count=25)` — t8452 분봉 OHLCV
- `get_total_capital()` — CSPAQ12300 예탁자산총액
- `get_balance()` — CSPAQ12300 보유종목 잔고 목록
- `place_order(code, qty, side, order_type)` — CSPAT00601 주문

**indicator_calc.py**
- `calc_atr(ohlcv_list, period=20)` — ATR(N)
- `calc_ma(close_list, period)` — 이동평균
- `calc_10day_low(ohlcv_list)` — 10일 신저가
- `get_all_indicators(code, ls_client)` — 한 번에 모두 계산 후 딕셔너리 반환

**trade_ledger.py** ← [보완 #2: KST 적용]
- `append_trade(record)` — JSON + Google Sheets 저장
- `_generate_record_id()` — 고유 ID 생성
- 모든 타임스탬프는 `pytz` KST 기준

**telegram_alert.py**
- `SendMessage(msg)` — 텔레그램 봇 발송 (1회 실패 시 1회 재시도 후 포기)

**target_manager.py** ← [보완 #2: KST 적용]
- `calc_pending_target(code, current_price, indicators)` — max(현재가×1.02, 240분봉20MA×1.005)
- `update_above_target_time(code, current_price, record)` — 목표가 이상이면 KST 타임스탬프 기록, 미달이면 null 초기화
- `run_update()` — 미보유 종목 전체 목표가 갱신

**timer_agent.py**
- `check_30min_passed(code, unheld_record)` — above_target_since로부터 30분 경과 확인
- `run_timer_check()` — 전체 체크 후 통과 종목 코드 리스트 반환

**turtle_order_logic.py** ← [보완 #3: 피라미딩 시 손절가 갱신 / 보완 #4: 종목 범위 체크]
- `calc_unit_size(code, price, atr_n, total_capital)` — Unit 수량 계산 (예외 진입 포함)
- `place_entry_order(code, qty, price, entry_type)` — 1차 진입 + position_state 저장
  - stop_loss_price = price - 2×atr_n
  - next_pyramid_price = price + 0.5×atr_n
- `place_pyramid_order(code, qty, price, atr_n)` — 피라미딩 + last_buy_price·stop_loss_price·next_pyramid_price 갱신
- `check_pyramid_trigger(code, current_price, pos_state, atr_n)` — 현재가 ≥ next_pyramid_price 확인
- `run_orders(entry_signals)` — 진입 신호 처리 + 피라미딩 체크
  - **모든 주문 전 `lovely_stock_list` 포함 여부 확인**

**risk_guardian.py** ← [보완 #1: 수익권 조건 / 보완 #4: 종목 범위 체크]
- `check_hard_stop(code, current_price, pos_state, atr_n)` — 현재가 ≤ last_buy_price - 2×atr_n
- `check_trailing_stop(code, current_price, pos_state, indicators)`:
  - 10일 신저가 경신 → 무조건 청산
  - 5MA 하향 돌파 → **current_price > 평균매입단가(수익권)일 때만** 청산
- `place_exit_order(code, qty, reason)` — 전량 매도 + position_state 삭제
  - **주문 전 `lovely_stock_list` 포함 여부 확인**
- `run_guardian()` — 전체 보유 종목 감시

**run_all.py** ← [보완 #5: 장외 시간 즉시 종료]
```
1. KST 기준 현재 시각 확인
   → 09:00 ~ 15:30 범위 밖이면 로그 남기고 즉시 종료
2. login()
3. risk_guardian.run_guardian()
4. target_manager.run_update()
5. entry_signals = timer_agent.run_timer_check()
6. turtle_order_logic.run_orders(entry_signals)
```
