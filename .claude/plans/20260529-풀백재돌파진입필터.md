## 🎯 목표

현재 터틀 전략은 20일 또는 55일 신고가를 돌파한 뒤 **30분이 지나면** 매수합니다.
이 방식은 "가짜 돌파" — 잠깐 신고가를 찍었다가 바로 내려오는 경우 — 에 취약합니다.

이번 작업은 30분 가드를 제거하고, 대신 **"눌림 후 재돌파"(풀백·재테스트)** 방식으로 진입 조건을 교체합니다.
주가가 신고가를 찍고 한 번 눌린 뒤 다시 그 최고점을 돌파할 때만 매수하도록 바꿔, 가짜 돌파에 걸리는 횟수를 줄이는 것이 목적입니다.

## 📖 확정된 전략 요약 (대화에서 결정된 내용)

### 새로운 진입 방식: 풀백 재돌파

주가가 신고가를 돌파한 뒤 세 단계를 거쳐야 매수합니다.

```
[1단계: WATCHING — 올라가는 중]
  조건: turtle_s1_signal = True (20일 신고가 돌파)
        OR turtle_s2_signal = True (55일 신고가 돌파)
        AND 현재 시각 >= 10:00
  동작: 실시간가가 올라가는 동안 최고값을 계속 갱신
  → 실시간가가 최고값보다 낮아지는 순간 2단계로 넘어감

[2단계: PULLBACK — 눌림 중]
  동작: 최고값을 잠금 (더 이상 갱신하지 않음)
  → 실시간가 > 최고값 → 매수!
  → 실시간가 < 돌파값 → 전체 초기화 (신호·최고값 모두 삭제)

[매수 실행]
  손절: 체결가 기준 2N (처음부터 끝까지 동일)
```

### 기존 대비 변경 요약

| 항목 | 기존 | 변경 후 |
|------|------|---------|
| 진입 필터 | 돌파 후 30분 경과 | 눌림 후 최고값 재돌파 |
| 손절 | 2N | 2N (동일, 변경 없음) |

### 초기화 규칙

- 실시간가가 돌파값(20일·55일 신고가) 아래로 내려가면 → **신호·최고값 모두 삭제**
- 다음에 다시 신고가를 돌파하면 → 새 사이클로 처음부터 시작

### JSON에 추가해야 할 필드 (`unheld_stock_record.json`)

| 필드명 | 설명 |
|--------|------|
| `turtle_s1_peak_price` | 20일 신고가 돌파 후 장중 최고값 |
| `turtle_s1_peak_locked` | 최고값 잠금 여부 (True = 눌림 시작됨) |
| `turtle_s2_peak_price` | 55일 신고가 돌파 후 장중 최고값 |
| `turtle_s2_peak_locked` | 최고값 잠금 여부 |
