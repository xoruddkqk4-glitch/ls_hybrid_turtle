## 🎯 목표

지금은 감시 종목(화이트리스트·블랙리스트)을 바꾸려면 PC에서 `watchlist_config.json` 파일을 직접 열어 수정해야 합니다.
이 계획은 **휴대폰 텔레그램 앱에서 명령어 한 줄**로 종목을 추가·제거할 수 있게 만드는 것입니다.

추가로 보유 종목·잔고·시스템 상태도 텔레그램으로 즉시 확인할 수 있게 합니다.

## 📦 결정된 사항

- **운영 방식**: Polling (텔레그램에서 메시지를 주기적으로 가져오기)
- **실행 위치**: AWS 서버에서 24시간 백그라운드 데몬으로 실행
- **공격 표면 최소화**: webhook(외부 포트 개방)이 아닌 polling으로 → 보안 위험 없음

#### 데이터 흐름도

```
[사용자 휴대폰 텔레그램 앱]
    ↓ "/add 005930 삼성전자"
[텔레그램 서버]
    ↓ getUpdates (10초 polling)
[telegram_listener.py 데몬]
    ├─ 권한 검증: chat.id == TELEGRAM_CHAT_ID ?
    │   └─ NO → 사용자에게 거부 메시지 + 관리자에게 침입 알림
    ├─ 명령어 파싱 ("/add" → handler_add)
    ├─ telegram_commands.handle_add(["005930", "삼성전자"])
    │   ├─ 인자 검증 (코드 6자리·이름 1자 이상)
    │   ├─ watchlist_writer.add_to_whitelist(...)
    │   │   ├─ watchlist_config.json 갱신 (임시파일→rename)
    │   │   └─ dynamic_watchlist.json 즉시 갱신
    │   └─ 결과 메시지 반환
    ├─ telegram_alert.send_reply(chat_id, "✅ 삼성전자(005930) 추가됨")
    └─ telegram_offset.json 갱신 (update_id+1 저장)
```

#### 안전 패턴

1. **원자적 쓰기**: `tempfile` → `os.replace()` 패턴 (Linux/Windows 모두 atomic)
2. **권한 검증 실패 시**:
   - 사용자 → "권한 없음" 거부 메시지
   - 관리자(`TELEGRAM_CHAT_ID`) → 침입 시도 알림 (chat_id, username, 메시지 본문)
3. **데몬 안정성**:
   - `getUpdates` 실패 → 지수 백오프(1·2·4·8·16·32·60초 상한)
   - 한 명령 처리 중 예외 → 사용자에겐 친절 메시지·관리자엔 트레이스백·데몬은 계속 실행
   - SIGTERM/SIGINT → offset 저장 후 종료
4. **단일 인스턴스 보장**: systemd `Restart=on-failure`, 인스턴스 1개 (같은 봇 토큰으로 두 데몬 띄우면 메시지 분산)

#### systemd 서비스 파일 (`deploy/ls_telegram_listener.service`)

```ini
[Unit]
Description=LS Hybrid Turtle Telegram Listener
After=network-online.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/var/autobot/ls_hybrid_turtle
ExecStart=/usr/bin/python3 /var/autobot/ls_hybrid_turtle/telegram_listener.py
Restart=on-failure
RestartSec=10
StandardOutput=append:/var/autobot/ls_hybrid_turtle/logs/telegram_listener.log
StandardError=append:/var/autobot/ls_hybrid_turtle/logs/telegram_listener.log

[Install]
WantedBy=multi-user.target
```

설치/실행:
```bash
sudo cp deploy/ls_telegram_listener.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now ls_telegram_listener
journalctl -u ls_telegram_listener -f   # 실시간 로그
```
